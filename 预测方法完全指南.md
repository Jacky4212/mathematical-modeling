## 1. 灰色预测 GM(1,1)

### 数学原理

GM(1,1) 意为"一阶单变量灰色模型"。核心思想：对非负原始序列作一次累加生成(1-AGO)，消除随机波动使序列呈近似指数规律，然后用一阶白化微分方程建模，最后累减还原(IAGO)得到预测值。

**建模步骤**：

- **步骤1** — 一次累加生成(1-AGO)：$x^{(1)}(k) = \sum_{i=1}^{k} x^{(0)}(i)$
- **步骤2** — 紧邻均值生成：$z^{(1)}(k) = 0.5[x^{(1)}(k) + x^{(1)}(k-1)]$
- **步骤3** — 灰微分方程：$x^{(0)}(k) + a z^{(1)}(k) = b$
- **步骤4** — 白化微分方程：$dx^{(1)}/dt + a x^{(1)} = b$
- **步骤5** — 最小二乘解 $[a,b]^T = (B^T B)^{-1} B^T Y$
- **步骤6** — 时间响应函数：$\hat{x}^{(1)}(k+1) = [x^{(0)}(1) - b/a]e^{-ak} + b/a$
- **步骤7** — 累减还原：$\hat{x}^{(0)}(k+1) = \hat{x}^{(1)}(k+1) - \hat{x}^{(1)}(k)$

**级比检验**：级比 $\lambda(k) = x^{(0)}(k-1)/x^{(0)}(k)$ 必须落在区间 $[e^{-2/(n+1)}, e^{2/(n+1)}]$ 内

### 适用场景
- 小样本预测（4~10个数据点即可）
- 信息不完备的"灰色系统"
- 近似指数增长的数据（GDP、电力负荷、污染物浓度）
- 中短期预测

### 不适用场景
- 数据波动剧烈、不呈指数趋势
- 数据有负值
- 长期预测（误差累积严重）

### 优缺点
| 优点 | 缺点 |
|------|------|
| 样本需求极小，4个点即可 | 只适合指数增长规律 |
| 不需要特定概率分布假设 | 对转折点和波动无能为力 |
| 计算简单，易于实现 | 长期预测精度急剧下降 |
| 指数型数据精度高 | \|a\| > 2 时模型失效 |

### 关键参数
| 参数 | 含义 | 取值范围 |
|------|------|----------|
| a（发展系数） | 系统发展态势 | a∈(-2,2); a<0增长, a>0衰减 |
| b（灰色作用量） | 外部环境驱动强度 | 任意实数 |

### 精度等级（MAPE）
- <1%：一级（优秀）
- <5%：二级（合格）
- <10%：三级（勉强合格）
- >=10%：四级（不合格）

### 实现注意事项
1. 数据必须非负，若有负值需做平移变换
2. 级比检验不通过时，加常数平移：$x' = x + c$
3. 只适合中短期预测，预测步数不宜超过样本数的1/3
4. 建模至少需要4个数据点

### 参考代码
```python
import numpy as np

def gm11(x0, predict_n=1):
    """GM(1,1)灰色预测"""
    x0 = np.array(x0, dtype=float)
    n = len(x0)

    # 1-AGO累加生成
    x1 = np.cumsum(x0)

    # 紧邻均值生成
    z1 = (x1[:-1] + x1[1:]) / 2.0

    # 最小二乘估计 [a, b]
    B = np.vstack([-z1, np.ones(n-1)]).T
    Y = x0[1:]
    a, b = np.linalg.lstsq(B, Y, rcond=None)[0]

    # 时间响应函数
    def predict(k):
        return (x0[0] - b/a) * np.exp(-a * k) + b/a

    # 拟合值
    x1_fit = np.array([predict(k) for k in range(n + predict_n)])
    x0_fit = np.diff(x1_fit, prepend=x0[0])

    # 预测值
    forecast = x0_fit[-predict_n:] if predict_n > 0 else np.array([])

    # 后验差检验
    residual = x0 - x0_fit[:n]
    C = residual.std() / x0.std()  # 后验差比值
    P = (np.abs(residual - residual.mean()) < 0.6745 * x0.std()).mean()  # 小误差概率

    return x0_fit[:n], forecast, {'a': a, 'b': b, 'C': C, 'P': P}

def grade_test(x0):
    """级比检验"""
    n = len(x0)
    lam = x0[:-1] / x0[1:]
    lower = np.exp(-2/(n+1))
    upper = np.exp(2/(n+1))
    valid = (lam >= lower) & (lam <= upper)
    return lam, (lower, upper), valid

# 使用示例
import numpy as np
x0 = np.array([2.67, 3.13, 3.25, 3.36, 3.56, 3.72])
fit, pred, stats = gm11(x0, predict_n=2)
print(f"拟合: {fit}")
print(f"预测: {pred}")
print(f"a={stats['a']:.4f}, 后验差比C={stats['C']:.4f}")
```

## 2. ARIMA 时间序列

### 数学原理

ARIMA(p,d,q) = 自回归(AR) + 差分(I) + 移动平均(MA)

- **AR(p)**：$y_t = c + \phi_1 y_{t-1} + \cdots + \phi_p y_{t-p} + \varepsilon_t$
- **I(d)**：差分使序列平稳，一阶 $\Delta y_t = y_t - y_{t-1}$
- **MA(q)**：$y_t = c + \varepsilon_t + \theta_1 \varepsilon_{t-1} + \cdots + \theta_q \varepsilon_{t-q}$

**ACF/PACF 定阶规则**：
| | ACF（自相关） | PACF（偏自相关） |
|------|--------------|------------------|
| AR(p) | 拖尾（衰减） | **p阶后截尾** |
| MA(q) | **q阶后截尾** | 拖尾（衰减） |
| ARMA(p,q) | 拖尾 | 拖尾 |

### 建模流程
1. ADF平稳性检验 → 不平稳则差分
2. 白噪声检验 → 若是白噪声则无需建模
3. 绘制ACF/PACF图 → 初步确定p、q范围
4. 网格搜索 + AIC/BIC最优选择
5. 残差诊断（Ljung-Box检验、Q-Q图）
6. 预测 + 置信区间

### 适用场景
- 平稳或可通过差分平稳化的单变量时间序列
- 无明显季节性的数据
- 经济指标（GDP、CPI）、销售预测、库存管理

### 优缺点
| 优点 | 缺点 |
|------|------|
| 理论基础坚实 | 无法捕捉非线性模式 |
| 提供预测区间 | 多步预测误差累积快 |
| 模型可解释性强 | 定阶过程有主观性 |

### 关键参数
| 参数 | 含义 | 建议范围 |
|------|------|----------|
| p | 自回归阶数 | 0~5 |
| d | 差分阶数 | 0~2（通常≤1） |
| q | 移动平均阶数 | 0~5 |

### 实现注意事项
1. ADF检验 p<0.05才认为平稳
2. AIC/BIC越小越好，BIC对复杂模型惩罚更重
3. 残差必须通过Ljung-Box白噪声检验（p>0.05）
4. 小样本时p、q不宜超过2

### 参考代码
```python
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.stattools import adfuller

def arima_forecast(series, order=(1,1,1), steps=5):
    """ARIMA建模与预测"""
    # ADF平稳性检验
    adf_result = adfuller(series.dropna())
    print(f"ADF统计量: {adf_result[0]:.4f}, p值: {adf_result[1]:.4f}")

    # 建模
    model = ARIMA(series, order=order)
    fitted = model.fit()

    # 预测
    forecast = fitted.forecast(steps=steps)
    return fitted, forecast

# 自动定阶（简化版）
def auto_arima_simple(series, p_range=(0,3), q_range=(0,3)):
    """网格搜索最优ARIMA(p,1,q)"""
    best_aic = np.inf
    best_order = None
    for p in range(p_range[0], p_range[1]+1):
        for q in range(q_range[0], q_range[1]+1):
            try:
                model = ARIMA(series, order=(p,1,q))
                fitted = model.fit()
                if fitted.aic < best_aic:
                    best_aic = fitted.aic
                    best_order = (p,1,q)
            except:
                continue
    return best_order
```

## 3. 季节性 ARIMA (SARIMA)

### 数学原理
SARIMA(p,d,q)(P,D,Q)s — 在ARIMA基础上增加季节性成分：

$$\Phi_P(B^s) \phi_p(B) (1-B)^d (1-B^s)^D y_t = \Theta_Q(B^s) \theta_q(B) \varepsilon_t$$

| 参数 | 含义 | 示例（月度数据s=12） |
|------|------|---------------------|
| p | 非季节性AR阶数 | $y_{t-1}, y_{t-2}$ |
| d | 非季节性差分 | $\Delta y_t$ |
| q | 非季节性MA阶数 | $\varepsilon_{t-1}$ |
| P | 季节性AR阶数 | $y_{t-12}, y_{t-24}$ |
| D | 季节性差分 | $\Delta_{12} y_t = y_t - y_{t-12}$ |
| Q | 季节性MA阶数 | $\varepsilon_{t-12}$ |
| s | 季节周期 | 12(月), 4(季), 7(周) |

### 适用场景
有明显季节性/周期性的数据：电力负荷、零售销售、旅游客流

### 优缺点
| 优点 | 缺点 |
|------|------|
| 能捕捉季节性模式 | 参数多（7个），调参复杂 |
| 统计性质明确 | 需要至少3-4个完整季节周期 |
| 自动搜索可用 | P+Q太大会过拟合 |

### 参考代码
```python
from statsmodels.tsa.statespace.sarimax import SARIMAX

# 月度数据 s=12
model = SARIMAX(series, order=(1,1,1), seasonal_order=(1,1,1,12))
fitted = model.fit()
forecast = fitted.forecast(steps=12)

# 或使用 pmdarima 自动搜索
# from pmdarima import auto_arima
# auto_model = auto_arima(series, seasonal=True, m=12, trace=True)
```

## 4. GARCH 模型（波动率建模）

### 数学原理

**均值方程**：$r_t = \mu + \varepsilon_t$，$\varepsilon_t = \sigma_t z_t, z_t \sim N(0,1)$

**方差方程 GARCH(p,q)**：
$$\sigma^2_t = \omega + \sum_{i=1}^{q} \alpha_i \varepsilon^2_{t-i} + \sum_{j=1}^{p} \beta_j \sigma^2_{t-j}$$

**最简形式 GARCH(1,1)**：$\sigma^2_t = \omega + \alpha \varepsilon^2_{t-1} + \beta \sigma^2_{t-1}$
- $\omega$：长期平均方差水平
- $\alpha$：新信息冲击程度（ARCH效应）
- $\beta$：波动率持续性（GARCH效应）
- $\alpha+\beta$：越接近1波动持续越久

### 扩展模型
| 模型 | 特点 |
|------|------|
| EGARCH | 捕捉非对称效应（杠杆效应） |
| GJR-GARCH | 负收益对波动影响更大 |
| GARCH-M | 波动率进入均值方程（风险溢价） |

### 适用场景
金融资产收益率波动率建模、VaR风险度量、期权定价

### 优缺点
| 优点 | 缺点 |
|------|------|
| 能捕捉波动聚集和尖峰厚尾 | 对称处理正负冲击（需扩展） |
| GARCH(1,1)简洁且效果足够 | 无法解释波动来源 |
| 理论基础扎实 | 多步预测波动率趋向无条件方差 |

### 实现注意事项
1. 收益率需缩放：过小会导致收敛问题，建议x100
2. 金融数据用't'或'skewt'分布优于'normal'
3. $\alpha+\beta < 1$（平稳性）；若 ≈1，考虑IGARCH
4. Python推荐库：`arch`

### 参考代码
```python
from arch import arch_model

returns = returns * 100  # 缩放避免收敛问题
model = arch_model(returns, vol='Garch', p=1, q=1, dist='t')
fitted = model.fit()
forecast = fitted.forecast(horizon=5)
```

## 5. VAR 向量自回归

### 数学原理
$$Y_t = c + \Pi_1 Y_{t-1} + \cdots + \Pi_p Y_{t-p} + \varepsilon_t$$

### 三大核心分析工具
1. **格兰杰因果检验**：检验X的滞后值是否能显著提高Y的预测精度
2. **脉冲响应函数(IRF)**：一个变量受到冲击后，各变量随后的动态响应
3. **方差分解(FEVD)**：量化不同冲击对预测误差方差的贡献

### 适用场景
多个相互影响的时间序列、宏观经济分析、货币政策传导

### 优缺点
| 优点 | 缺点 |
|------|------|
| 不预设变量的内生/外生性 | 参数多：$k^2 p$个参数 |
| 提供丰富分析工具(IRF/FEVD) | "维度诅咒" |
| 预测通常优于单变量 | IRF正交化依赖变量排序 |

### 参考代码
```python
from statsmodels.tsa.api import VAR

model = VAR(data)
lag_order = model.select_order(maxlags=12)
fitted = model.fit(lag_order.aic)
# 预测 + 脉冲响应 + 方差分解
forecast = fitted.forecast(data.values[-fitted.k_ar:], steps=5)
irf = fitted.irf(10)
```

## 6. 马尔可夫预测

### 数学原理
**马尔可夫性质**：$P(X_{n+1}=x | X_0,...,X_n) = P(X_{n+1}=x | X_n)$

**状态转移矩阵 P**：$P_{ij} = P(X_{n+1}=j | X_n=i)$，$\sum_j P_{ij} = 1$

**n步转移**：$P^{(n)} = P^n$

**稳态分布 $\pi$**：$\pi P = \pi, \sum_i \pi_i = 1$

### 适用场景
天气/信用等级预测、市场占有率分析、用户行为转换、排队系统

### 优缺点
| 优点 | 缺点 |
|------|------|
| 模型简单直观 | 无记忆假设常被违反 |
| 计算高效（矩阵运算） | 状态划分主观 |
| 稳态分析洞察长期行为 | 无法处理连续变量 |

### 参考代码
```python
def markov_predict(P, current_state, steps=1):
    """马尔可夫预测"""
    P = np.array(P)
    # n步转移矩阵
    P_n = np.linalg.matrix_power(P, steps)
    return P_n[current_state]

def steady_state(P):
    """计算稳态分布"""
    n = P.shape[0]
    # 解 pi P = pi, sum(pi)=1
    A = np.vstack([(P.T - np.eye(n))[:-1], np.ones(n)])
    b = np.zeros(n)
    b[-1] = 1
    return np.linalg.lstsq(A, b, rcond=None)[0]
```

## 7. 多项式拟合（最小二乘）

### 数学原理
$$y = a_0 + a_1x + a_2x^2 + \cdots + a_mx^m + \varepsilon$$

**OLS**：$\min_\theta \sum (y_i - f(x_i;\theta))^2$，$\theta = (X^T X)^{-1} X^T y$

**Ridge(L2)**：$\min_\theta (\sum(y_i - f(x_i))^2 + \alpha\sum\theta_j^2)$

**Lasso(L1)**：$\min_\theta (\sum(y_i - f(x_i))^2 + \alpha\sum|\theta_j|)$

### Ridge vs Lasso
| 特性 | Ridge (L2) | Lasso (L1) |
|------|-----------|-----------|
| 特征选择 | 不归零 | 可压缩到0 |
| 适用 | 精度优先 | 特征维度高时 |

### 适用场景
数据呈曲线趋势但机制不明、趋势外推（短期）、插值、基线对比

### 优缺点
| 优点 | 缺点 |
|------|------|
| 理论简单，计算快速 | 高阶极易过拟合 |
| 闭式解，无需迭代 | 外推能力极差 |
| Ridge/Lasso提供正则化 | 无法捕捉周期性和突变 |

### 参考代码
```python
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.linear_model import Ridge, Lasso
from sklearn.pipeline import Pipeline

# Ridge多项式拟合
degree = 3
pipe = Pipeline([
    ('poly', PolynomialFeatures(degree=degree)),
    ('scaler', StandardScaler()),
    ('ridge', Ridge(alpha=1.0))
])
pipe.fit(X, y)
y_pred = pipe.predict(X_test)
```

## 8. Logistic 模型（S曲线）

### 数学原理
**Logistic微分方程（Verhulst, 1838）**：
$$dP/dt = rP(1 - P/K)$$

**解析解（S曲线）**：
$$P(t) = \frac{K P_0 e^{rt}}{K + P_0(e^{rt} - 1)}$$

参数：K=环境容量，P0=初始值，r=内在增长率

**拐点**：$P=K/2$ 时增长率最大

### 适用场景
人口增长预测、传染病累计感染、市场渗透率/产品扩散、技术S曲线

### 不适用场景
无上限增长、多峰/多波传播、受多重外部冲击的系统

### 优缺点
| 优点 | 缺点 |
|------|------|
| 有生物学/经济学理论基础 | 对称假设：实际常不对称 |
| 参数含义直观 | 无法解释多波/多峰 |
| 仅3个参数 | 拐点固定(P=K/2) |

### 扩展模型
- **Richards曲线**：广义Logistic，增加形状参数m
- **Gompertz曲线**：拐点更早(P=K/e处)

### 参考代码
```python
from scipy.optimize import curve_fit

def logistic(t, K, P0, r):
    return K * P0 * np.exp(r * t) / (K + P0 * (np.exp(r * t) - 1))

popt, _ = curve_fit(logistic, t_data, y_data,
                     p0=[max(y_data)*1.5, y_data[0], 0.3],
                     maxfev=10000)
K, P0, r = popt
y_pred = logistic(t_future, K, P0, r)
```

## 9. 双重差分 DID

### 数学原理
$$Y_{it} = \alpha + \gamma \text{Treated}_i + \lambda \text{Post}_t + \delta (\text{Treated}_i \times \text{Post}_t) + \varepsilon_{it}$$

$$\text{ATT} = (\bar{Y}_{T,Post} - \bar{Y}_{T,Pre}) - (\bar{Y}_{C,Post} - \bar{Y}_{C,Pre})$$

### 平行趋势假设（核心）
若无政策干预，处理组和对照组应有相同的时间变化趋势。

**检验方法**：画图法、事件研究法、安慰剂检验

### 适用场景
政策/项目实施效果评估、自然实验、有处理组和对照组的面板数据

### 优缺点
| 优点 | 缺点 |
|------|------|
| 概念直观 | 平行趋势假设不可直接检验 |
| 控制固定差异+共同趋势 | 对对照组质量高度依赖 |
| 回归框架简洁 | 无法处理时变混淆因素 |

### 交错处理现代方法
- Callaway & Sant'Anna (2021)：分组-时间ATT
- Sun & Abraham (2021)：交互加权估计器
- Borusyak et al. (2024)：填补法

### 参考代码

## 10. PSM 倾向得分匹配

### 数学原理
为每个处理组个体在对照组中找倾向得分最接近的"替身"，模拟随机对照实验。

**倾向得分**：$e(X) = P(T=1 | X)$ — 给定协变量后接受处理的概率

**CIA假设**：$Y(0), Y(1) \perp T | X$

### 三步法
| 步骤 | 内容 | 方法 |
|------|------|------|
| Step 1 | 倾向得分估计 | Logit/Probit回归 |
| Step 2 | 匹配 | 最近邻/卡尺/核匹配 |
| Step 3 | ATT估计 | 匹配后结局差异 |

### 匹配方法
| 方法 | 特点 |
|------|------|
| 最近邻1:1 | 简单，样本损失少 |
| 最近邻1:k | 降低方差 |
| 卡尺匹配 | 限制得分差≤ε，质量更高 |
| 核匹配 | 全部对照加权，利用所有信息 |

### 平衡性检验
标准化均值差 |SMD| < 0.1（理想），< 0.2（可接受）

### 适用场景
政策效果评估（非随机分配）、医学治疗效果对比、营销活动归因

### 优缺点
| 优点 | 缺点 |
|------|------|
| 概念直观 | 只能控制可观测混淆 |
| 不依赖函数形式假设 | 对倾向得分模型误设敏感 |
| 平衡性检验验证质量 | 匹配过程有主观性 |

### 参考代码
```python
from sklearn.linear_model import LogisticRegression

# 倾向得分估计
ps_model = LogisticRegression()
ps_model.fit(X, T)
ps = ps_model.predict_proba(X)[:, 1]

# 最近邻匹配（简化版）
from sklearn.neighbors import NearestNeighbors

treated_ps = ps[T == 1].reshape(-1, 1)
control_ps = ps[T == 0].reshape(-1, 1)
nn = NearestNeighbors(n_neighbors=1)
nn.fit(control_ps)
distances, indices = nn.kneighbors(treated_ps)

# ATT
att = (Y[T == 1] - Y[T == 0][indices.flatten()]).mean()
```

## 11. BP 神经网络预测

### 数学原理
**万能逼近定理**：单隐藏层+Sigmoid+线性输出，隐藏层神经元足够多，可任意精度逼近任何Borel可测函数。

**正向传播**：
- 隐层：$h = \sigma(W_1 x + b_1)$
- 输出：$\hat{y} = W_2 h + b_2$

**反向传播**：链式法则计算梯度 → 梯度下降更新权重

### 激活函数对比
| 函数 | 公式 | 特点 |
|------|------|------|
| Sigmoid | $1/(1+e^{-x})$ | 输出(0,1)，梯度消失 |
| Tanh | $(e^x-e^{-x})/(e^x+e^{-x})$ | 输出(-1,1)，零中心 |
| ReLU | $\max(0,x)$ | 最快，缓解梯度消失 |
| LeakyReLU | $\max(0.01x,x)$ | 解决"死神经元" |

### 过拟合应对
1. **Dropout**：训练时随机丢弃神经元
2. **Early Stopping**：验证误差不降时停止
3. **L2正则化**：权重衰减
4. **Batch Normalization**：加速训练+轻微正则化

### 适用/不适用场景
✅ 非线性复杂预测、多输入多输出、模式识别
❌ 小样本、需要强可解释性

### 参考代码
```python
import torch
import torch.nn as nn

class BPNet(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim=1):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.ReLU(),
            nn.Linear(hidden_dim // 2, output_dim)
        )

    def forward(self, x):
        return self.net(x)
```

## 12. 机器学习预测集成

### 四种算法核心对比

| 维度 | 随机森林 | XGBoost | LightGBM | CatBoost |
|------|:---:|:---:|:---:|:---:|
| 集成方式 | Bagging（并行） | Boosting（串行） | Boosting（串行） | Boosting（串行） |
| 树生长 | 独立 | Level-wise | **Leaf-wise** | Symmetric |
| 核心创新 | Bootstrap+随机特征 | 二阶泰勒+正则化 | GOSS+EFB | Ordered Boosting |
| 类别特征 | 需编码 | 需编码 | 原生支持 | **最强支持** |
| 缺失值 | 需填充 | 自动处理 | 自动处理 | 自动处理 |
| 训练速度 | ★★★★ | ★★★ | ★★★★★ | ★★★ |
| 预测精度 | ★★★ | ★★★★★ | ★★★★★ | ★★★★★ |
| 调参难度 | ★(简单) | ★★★★(复杂) | ★★★ | ★★ |
| 默认表现 | ★★★ | ★★★★ | ★★★ | ★★★★★ |

### 选型决策
| 场景 | 首选 |
|------|------|
| 快速基线/高鲁棒性 | 随机森林 |
| 追求极致精度 | XGBoost |
| 大数据集(>10万行) | LightGBM |
| 大量类别特征 | CatBoost |
| 最少调参/开箱即用 | CatBoost |

### 关键参数调优顺序
- **随机森林**：n_estimators → max_depth → min_samples_split → max_features
- **XGBoost**：n_estimators+lr → max_depth → subsample → reg_alpha/lambda
- **LightGBM**：num_leaves(核心) → max_depth → lr → feature_fraction → min_data_in_leaf
- **CatBoost**：iterations+lr → depth → l2_leaf_reg → random_strength

### 按数据特征
| 数据特征 | 推荐方法 | 原因 |
|----------|----------|------|
