12. 排队论（随机服务系统理论）

返回首页 →

一、排队论概述

排队论（Queueing Theory）

又名随机服务系统理论，是运筹学和应用概率的重要分支。研究拥挤现象——通过分析各种服务系统在排队等待中的概率特性，来解决系统的最优设计和最优控制问题。起源于20世纪初丹麦电信工程师A.K.Erlang对电话系统的研究。

1.1 生活中的排队系统

顾客要求的服务服务台
储户银行储蓄出纳员/ATM
病人就诊医生
故障机器修理维修工
飞机降落跑道
货轮装卸货码头
车辆通过路口红绿灯

1.2 排队系统三要素

输入过程 → 排队规则 → 服务台
顾客到达 → （排队等待）→ 接受服务 → 离开
  • 输入过程：顾客源（有限/无限）、到达方式（单个/成批）、间隔时间分布
  • 排队规则：损失制 / 等待制(FCFS/LCFS/随机/优先权) / 混合制
  • 服务台：个数（1/多个/串联/并联）、服务方式、服务时间分布

二、Kendall记号与数量指标

2.1 排队模型的符号表示

$$X / Y / Z / A / B / C$$
符号含义常见取值
X到达间隔时间分布M(负指数), D(定长), Eₖ(k阶Erlang), G(一般)
Y服务时间分布同上
Z服务台个数1, 2, ..., c
A系统容量限制N（默认∞）
B顾客源数目m（默认∞）
C服务规则FCFS（默认）/ LCFS / PS / SIRO
简化记号：略去后三项表示 X/Y/Z/∞/∞/FCFS。如 M/M/1 = M/M/1/∞/∞/FCFS

2.2 关键数量指标

符号含义关系
L平均队长（系统中全部顾客数）L = λW
Lq平均排队长（队列中等待顾客数）Lq = λWq
W平均逗留时间（等待+服务）W = Wq + 1/μ
Wq平均等待时间—
ρ服务强度 λ/μ（或λ/(cμ)）必须 ρ < 1 系统才稳定
B / I忙期 / 闲期反映服务台工作强度

三、常见概率分布

3.1 Poisson过程（M — 到达分布）

$$P\{N(t)=k\} = \frac{(\lambda t)^k}{k!} e^{-\lambda t}, \quad k=0,1,2,...$$

特性：平稳性、独立性、普通性。E[N(t)] = D[N(t)] = λt。λ为单位时间平均到达顾客数。

3.2 负指数分布（M — 间隔/服务分布）

$$f(t) = \lambda e^{-\lambda t}\ (t \geq 0), \quad E(T)=\frac{1}{\lambda}, \quad D(T)=\frac{1}{\lambda^2}$$
唯一具有Markov性（无记忆性）的连续分布：
P(T > t₀+t | T > t₀) = P(T > t)。这意味着不管已服务了多久，剩余服务时间仍服从同样的分布。

3.3 k阶Erlang分布（Eₖ）

$$f_k(t) = \frac{(k\mu)^k t^{k-1}}{(k-1)!} e^{-k\mu t}, \quad E(T)=\frac{1}{\mu}, \quad D(T)=\frac{1}{k\mu^2}$$

桥梁作用：k=1 → 负指数分布；k→∞ → 定长分布。Erlang分布可描述介于完全随机和完全确定之间的服务时间。

3.4 定长分布（D）

无任何随机波动，顾客等间隔到达或服务时间恒定。F(t)=1(t≥α), 0(t<α)。E(T)=α, D(T)=0。

3.5 分布检验：χ²检验

验证实际数据是否服从泊松分布：计算样本均值λ̄ → 求理论频数 e_n = N·P_n → 计算χ² = Σ(f_n−e_n)²/e_n → 查χ²临界值判断。

四、生灭过程（Birth-Death Process）

核心工具

处理输入为Poisson流、服务时间为负指数分布的排队模型的标准方法。"生"=顾客到达，"灭"=顾客离去。

稳态条件（转入率 = 转出率）：

$$\lambda_{n-1}p_{n-1} + \mu_{n+1}p_{n+1} = \lambda_n p_n + \mu_n p_n$$
$$\mu_1 p_1 = \lambda_0 p_0$$

稳态概率通解：

$$p_n = \frac{\lambda_{n-1}\lambda_{n-2}\cdots\lambda_0}{\mu_n\mu_{n-1}\cdots\mu_1} p_0, \quad p_0 = \frac{1}{1 + \sum_{n=1}^{\infty}\frac{\lambda_{n-1}\cdots\lambda_0}{\mu_n\cdots\mu_1}}$$

五、M/M/1 单服务台模型（核心）

5.1 假设条件

  • 到达：参数为λ的Poisson流（λ_n = λ常数）
  • 服务时间：参数为μ的负指数分布（μ_n = μ常数）
  • 单服务台，系统容量无限，FCFS排队规则
  • ρ = λ/μ < 1（到达率小于服务率，系统才能稳定）

5.2 核心公式

$$p_0 = 1-\rho, \quad p_n = \rho^n(1-\rho)$$
$$L = \frac{\lambda}{\mu-\lambda} = \frac{\rho}{1-\rho}$$
$$L_q = \frac{\rho^2}{1-\rho} = \frac{\lambda^2}{\mu(\mu-\lambda)}$$
$$W = \frac{1}{\mu-\lambda}, \quad W_q = \frac{\lambda}{\mu(\mu-\lambda)}$$
逗留时间分布：W 服从参数为 μ−λ 的负指数分布
f(t) = (μ−λ)e^{−(μ−λ)t}, F(t) = 1−e^{−(μ−λ)t} (t≥0)

六、其他排队模型速查

6.1 M/M/1/N（容量有限）

系统最多容纳N个顾客（含正在服务的），超出则损失。

$$p_n = \rho^n \cdot \frac{1-\rho}{1-\rho^{N+1}}, \quad n=0,1,...,N$$

6.2 M/M/c（多服务台并联）

c个并联服务台

$\rho = \lambda/(c\mu) < 1$，$\mu_n = \min(n,c)\cdot\mu$

$$p_0 = \left[\sum_{k=0}^{c-1}\frac{(c\rho)^k}{k!} + \frac{(c\rho)^c}{c!(1-\rho)}\right]^{-1}$$
$$L_q = \frac{(c\rho)^c \rho}{c!(1-\rho)^2} \cdot p_0, \quad L = L_q + c\rho$$

6.3 M/G/1（一般服务时间）

服务时间服从一般分布G，均值为1/μ，方差为σ²。

Pollaczek-Khintchine（P-K）公式
$$L_q = \frac{\lambda^2\sigma^2 + \rho^2}{2(1-\rho)}$$

6.4 各模型对比

模型Lq公式特点
M/M/1$\frac{\rho^2}{1-\rho}$基准模型
M/D/1$\frac{\rho^2}{2(1-\rho)}$定长服务，Lq为M/M/1的一半
M/Eₖ/1$\frac{(1+k)\rho^2}{2k(1-\rho)}$介于M/M/1和M/D/1之间

七、排队系统优化

7.1 最优服务率（M/M/1）

目标：min C(μ) = C_w·L(μ) + C_s·μ，其中C_w为单位等待成本，C_s为单位服务成本。

$$\mu^* = \lambda + \sqrt{\frac{C_w\lambda}{C_s}}$$

7.2 最优服务台数（M/M/c）

递增计算c=1,2,...直到 C(c+1) > C(c)：

$$\min_c \ C(c) = C_w \cdot L(c) + C_s \cdot c$$
典型例题：工地发放工具。λ=1.6人/分，μ=0.9人/分，等待费6元/时，空闲费3元/时。求最优服务窗口数。
ρ=1.6/0.9≈1.78 > 1 → 一个窗口不够 → 需c≥2个。分别计算c=2,3,4的Wq和总成本。

八、数学建模竞赛中的应用

排队论的竞赛应用场景：
1. 服务窗口设计：银行/医院/收费站窗口数量优化
2. 呼叫中心：客服人员排班、电话线路容量
3. 交通工程：红绿灯配时、收费站车道数
4. 生产管理：流水线工位设计、维修工人数
5. 通信网络：带宽分配、路由器缓冲区大小
6. 应急管理：救护车/消防车部署数量

Python实现（M/M/1和M/M/c）

import numpy as np from math import factorial def mm1(lam, mu): """M/M/1模型""" rho = lam / mu assert rho < 1, '系统不稳定：ρ >= 1' p0 = 1 - rho L = rho / (1 - rho) Lq = rho**2 / (1 - rho) W = 1 / (mu - lam) Wq = lam / (mu * (mu - lam)) return {'rho': rho, 'p0': p0, 'L': L, 'Lq': Lq, 'W': W, 'Wq': Wq} def mmc(lam, mu, c): """M/M/c模型""" rho = lam / (c * mu) assert rho < 1, '系统不稳定：ρ >= 1' # p0 s = sum((c*rho)**k / factorial(k) for k in range(c)) p0 = 1 / (s + (c*rho)**c / (factorial(c) * (1-rho))) Lq = (c*rho)**c * rho / (factorial(c) * (1-rho)**2) * p0 L = Lq + c * rho Wq = Lq / lam W = L / lam return {'rho': rho, 'p0': p0, 'L': L, 'Lq': Lq, 'W': W, 'Wq': Wq}
