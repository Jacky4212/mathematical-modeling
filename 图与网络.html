1. 图与网络基础概念

📖 图 (Graph)

图 $G = (V, E)$ 由顶点集 $V = \{v_1, v_2, \dots, v_n\}$ 和边集 $E = \{e_1, e_2, \dots, e_m\}$ 组成。每条边连接两个顶点，可带方向（有向图）或不带方向（无向图），可带权重（赋权图）。

1.1 图的分类

类型特点典型应用
无向图边无方向，$(u,v)$ 和 $(v,u)$ 是同一对道路网络、社交关系
有向图边有方向，$(u \to v)$ 单向通行交通流向、任务依赖
赋权图每条边有数值权重 $w(e)$距离、成本、时间
连通图任意两顶点间存在路径—
树连通无环图，$m = n-1$层级结构、最优布线
二分图顶点可分为两部分，边只连接不同部分的顶点任务分配、匹配问题

1.2 图的存储方式

方法空间适用
邻接矩阵$O(n^2)$稠密图、快速判断连通性
邻接表$O(n+m)$稀疏图、遍历邻接点
边列表$O(m)$Kruskal 等需要按边排序的算法

2. 最短路问题 竞赛高频

📌 问题描述：在赋权图中，找从起点 $s$ 到终点 $t$ 的权重之和最小的路径。

2.1 Dijkstra 算法（非负权）

核心思想：贪心 + 松弛。每次从未确定最短路的顶点中选 $d[v]$ 最小的标记为"已确定"，并更新其邻居的距离。

算法步骤
  1. 初始化 $d[s]=0$，其他 $d[v]=\infty$；全部顶点未标记
  2. 选未标记中 $d[v]$ 最小的 $v$，标记为已确定
  3. 对 $v$ 的每条出边 $(v, u, w)$：若 $d[v] + w < d[u]$，则 $d[u] = d[v] + w$
  4. 重复 2-3 直到所有顶点已确定或 $t$ 被标记

复杂度：朴素 $O(n^2)$，二叉堆优化 $O((n+m)\log n)$。

2.2 Floyd 算法（全源最短路）

核心思想：动态规划。$d[i][j]^{(k)}$ = 只经过前 $k$ 个顶点的 $i$ 到 $j$ 最短距离。

$$d[i][j] = \min(d[i][j],\; d[i][k] + d[k][j])$$

复杂度：$O(n^3)$。适合 $n \leq 200$、需要所有点对距离的场景。

算法适用复杂度
Dijkstra非负权，单源$O(n^2)$ / $O((n+m)\log n)$
Floyd任意权（无负环），全源$O(n^3)$
Bellman-Ford可含负权边，单源$O(nm)$
SPFABellman-Ford 的队列优化$O(km)$，最坏 $O(nm)$

3. 最小生成树 (MST) 必掌握

📌 问题描述：在连通赋权无向图中，找一棵包含所有顶点的树，使边的权重之和最小。典型场景：最优布线、管网设计、聚类。

3.1 Kruskal 算法（边贪心）

将所有边按权重升序排列，依次取边——若该边的两端点不在同一连通分量中（用并查集判断），则加入生成树。

复杂度：$O(m \log m)$（排序主导）。适合稀疏图。

3.2 Prim 算法（点贪心）

从任意顶点出发，每次选连接已选集合和未选集合的最小权重边，将其加入。

复杂度：$O(n^2)$ / $O((n+m)\log n)$。适合稠密图。

算法策略适合复杂度
Kruskal边贪心 + 并查集稀疏图$O(m\log m)$
Prim点贪心 + 优先队列稠密图$O((n+m)\log n)$

4. 网络流 竞赛重点

4.1 最大流问题

📌 问题描述：在有向图中，每条边有容量 $c(e)$。从源点 $s$ 到汇点 $t$，在不超过边容量的前提下，最大能输送多少流量？

Ford-Fulkerson 方法（增广路）：不断在残余网络中找 $s \to t$ 的路径，沿路径增加流量，直到不存在增广路。

  • Edmonds-Karp：用 BFS 找最短增广路，复杂度 $O(n m^2)$
  • Dinic：分层图 + 多路增广，复杂度 $O(n^2 m)$，实际很快

4.2 最小费用最大流

在最大流基础上，每条边还有单位费用。求在流量最大的前提下费用最小的流。

方法：用 SPFA/Dijkstra 代替 BFS 找费用最小的增广路。

4.3 典型应用

应用建模方式
二分图最大匹配源连左部，右部连汇，容量 1
最小割最大流 = 最小割容量
任务分配源→任务→工人→汇
运输调度源→仓库→中转→需求点→汇

5. 关键路径 (CPM) 项目管理必用

📌 问题描述：给定一系列有先后依赖关系的任务（AOV 网），找决定整个项目周期的最长路径（关键路径）。关键路径上的任务不能延误。

计算步骤：

  1. 对 AOV 网做拓扑排序，确保每个任务的前驱已计算
  2. 顺推：计算每个任务的最早开始时间 $ES[v] = \max(ES[u] + w(u,v))$
  3. 逆推：计算最晚开始时间 $LS[u] = \min(LS[v] - w(u,v))$
  4. 松弛时间：$slack = LS[v] - ES[v]$。$slack = 0$ 的顶点构成关键路径

6. 旅行商 (TSP) 与 中国邮递员问题

6.1 旅行商问题 (TSP)

NP-Hard 经典问题

给定 $n$ 个城市和两两之间的距离，找一条经过每个城市恰好一次且回到起点的最短回路。

竞赛实用方法：

方法特点
贪心 + 2-opt 局部优化简单有效，$n \leq 100$ 可用
遗传算法 / 模拟退火$n \leq 200$，论文推荐
动态规划 (Held-Karp)$O(n^2 2^n)$，$n \leq 20$
LKH 启发式$n$ 可达数千，工业级

6.2 中国邮递员问题 (CPP)

区别于 TSP

邮递员从邮局出发，每条街道至少走一次，回到邮局的最短回路。本质是在图中添加边使其变为欧拉图。

解法：找出所有奇数度顶点 → 求它们之间的最短路径（Floyd）→ 做最小权完美匹配 → 将匹配路径对应的边"加倍" → 在欧拉图上找欧拉回路。

7. 二分图与匹配问题

7.1 最大匹配

📌 问题描述：二分图 $G = (L \cup R, E)$，找最多的边使得每个顶点最多属于一条匹配边。

匈牙利算法 (Hungarian)：基于增广路的贪心算法，复杂度 $O(nm)$。DFS 找增广路 → 反转路径上的匹配状态 → 匹配数 +1。

7.2 最优匹配 / 指派问题

每条匹配边有代价，找代价最小的完美匹配。

Kuhn-Munkres 算法 (KM / Hungarian)：复杂度 $O(n^3)$。通过调整顶点标号（potential）使相等子图逐渐扩大，直到找到完美匹配。

8. 竞赛建模案例

8.1 加油车调度问题

问题描述

多个加油站有订单需求，加油车从车场出发，在满足各油站需求的前提下使运输总距离最短。涉及：

  1. 车辆路径问题 (VRP)：多辆车、多站点、容量约束 → 最短路 + 分配 + TSP 的组合
  2. 紧急调度（油价上涨/地震/洪水）：加时间窗约束 → VRPTW
  3. 车辆检测：部分车辆下线检测 → 动态调度 / 重新分配

8.2 建模思路

  1. 阶段一（日常调度）：以各油站需求为顶点，车场为源点，建立带容量约束的 VRP 模型 → 聚类分配油站 → 每辆车跑 TSP
  2. 阶段二（紧急事件）：加入时间窗约束 $[a_i, b_i]$，优先满足紧急需求 → 带时间窗的 VRP (VRPTW)
  3. 阶段三（车辆检测）：下线车辆对应的路线重新分配给其他车辆 → 增量式更新方案

9. Python 代码实现

9.1 Dijkstra + Floyd

import heapq, numpy as np # Dijkstra 单源最短路 (堆优化) def dijkstra(graph, start, n): dist = [float('inf')] * n dist[start] = 0 pq = [(0, start)] while pq: d, u = heapq.heappop(pq) if d > dist[u]: continue for v, w in graph[u]: if dist[u] + w < dist[v]: dist[v] = dist[u] + w heapq.heappush(pq, (dist[v], v)) return dist # Floyd 全源最短路 def floyd(adj, n): d = adj.copy() for k in range(n): for i in range(n): for j in range(n): d[i][j] = min(d[i][j], d[i][k] + d[k][j]) return d

9.2 MST (Kruskal)

# Kruskal 最小生成树 class DSU: def __init__(self, n): self.p = list(range(n)) def find(self, x): if self.p[x] != x: self.p[x] = self.find(self.p[x]) return self.p[x] def union(self, a, b): self.p[self.find(a)] = self.find(b) def kruskal(edges, n): edges.sort(key=lambda e: e[2]) dsu, mst, total = DSU(n), [], 0 for u, v, w in edges: if dsu.find(u) != dsu.find(v): dsu.union(u, v) mst.append((u, v, w)) total += w return mst, total

9.3 匈牙利算法 (二分图匹配)

def hungarian(adj, n_left, n_right): match_r = [-1] * n_right def dfs(u, visited): for v in adj[u]: if not visited[v]: visited[v] = True if match_r[v] == -1 or dfs(match_r[v], visited): match_r[v] = u return True return False result = 0 for u in range(n_left): if dfs(u, [False]*n_right): result += 1 return result, match_r

常见陷阱

陷阱后果对策
负权边用 Dijkstra结果错误有负权边用 Bellman-Ford / SPFA
