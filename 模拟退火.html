11. 模拟退火算法

排队论 →

一、算法简介

1.1 起源与思想

物理退火 → 算法模拟

固体退火：将金属加热至高温 → 粒子自由运动（高能态）→ 缓慢降温 → 粒子趋于有序排列（低能态/全局最优）。模拟退火算法(Simulated Annealing, SA)由Kirkpatrick于1983年引入优化领域，模拟这一物理过程来求解组合优化问题。

物理退火模拟退火算法
粒子状态问题的解（状态空间中的点）
能量最低态全局最优解
加温过程设定较高的初始温度T₀
等温过程Metropolis采样过程（内循环）
冷却过程控制参数T下降（外循环）
系统能量目标函数值 f(x)

1.2 为什么要用模拟退火

核心优势：概率性跳出局部最优。传统贪心/爬山算法只接受更优解 → 陷入局部最优。模拟退火在高温时以一定概率接受更差解 → 有机会跳出局部最优 → 随着温度降低逐渐收敛到全局最优。

二、核心机制

2.1 Metropolis接受准则

接受概率

从当前解x产生新解x'，计算目标函数增量 Δf = f(x') − f(x)：

  • 若 Δf < 0（新解更优）→ 必定接受
  • 若 Δf ≥ 0（新解更差）→ 以概率 P = exp(−Δf / T) 接受
关键洞察：T越大 → exp(−Δf/T)越接近1 → 越容易接受差解（探索能力强）。T越小 → 接受概率趋于0 → 只接受更优解（趋于局部搜索）。这正是"退火"的精髓。

2.2 冷却调度（Cooling Schedule）

降温方式公式特点
指数降温（最常用）T_{k+1} = α·T_k, α∈[0.9, 0.99]前期快后期慢，广泛应用
对数降温T_k = T₀/ln(1+k)收敛有理论保证但太慢
线性降温T_{k+1} = T_k − ΔT匀速降温，简单但效果一般

每个温度下进行L次迭代（Markov链长度），确保在该温度下充分搜索。

三、算法流程

// 模拟退火算法伪代码 x = x0; // 初始解 T = T0; // 初始温度 while T > T_min: for i = 1 to L: // 内循环 (Markov链) x_new = neighbor(x) // 产生邻域新解 Δf = f(x_new) - f(x) if Δf < 0 or exp(-Δf/T) > random(0,1): x = x_new // 接受新解 T = α * T // 降温 return x

关键参数选取

参数建议值影响
初始温度 T₀使初始接受率≈0.8~0.95太高浪费计算，太低易陷入局部最优
降温率 α0.9~0.99越接近1精度越高但耗时
内循环长度 L100~1000×问题规模保证该温度下达到热平衡
终止温度 T_min10⁻⁶~10⁻³确保最终收敛到稳定解

四、经典应用：TSP旅行商问题

4.1 问题描述

给定N个城市的经纬度坐标，找到一条经过每个城市恰好一次并返回起点的最短路径。

4.2 模拟退火求解设计

设计要素实现
解的表示城市访问序列 path = [1, c₁, c₂, ..., c_N, 1]
初始解随机排列 + 选取最短的（如用randperm生成1000个取最优）
邻域操作（2-opt）随机选两点c₁,c₂，反转两位置之间的子路径
距离计算球面距离（Haversine公式）：d = 6370·acos(...)
目标函数路径总长度

4.3 MATLAB实现（PPT原代码）

% 距离矩阵（球面距离） d = zeros(102); for i = 1:101 for j = i+1:102 d(i,j) = 6370 * acos(cos(xy(i,1)-xy(j,1)) * cos(xy(i,2))... * cos(xy(j,2)) + sin(xy(i,2)) * sin(xy(j,2))); end end d = d + d'; % 对称化 % 模拟退火主循环 e = 0.1^30; L = 20000; at = 0.999; T = 1; for k = 1:L % 外循环：退火 c = 2 + floor(100 * rand(1,2)); % 随机选两个位置 c = sort(c); c1 = c(1); c2 = c(2); df = d(path(c1-1),path(c2)) + d(path(c1),path(c2+1))... - d(path(c1-1),path(c1)) - d(path(c2),path(c2+1)); if df < 0 % 新路径更短 → 接受 path = [path(1:c1-1), path(c2:-1:c1), path(c2+1:102)]; long = long + df; elseif exp(-df/T) >= rand % Metropolis准则 → 接受 path = [path(1:c1-1), path(c2:-1:c1), path(c2+1:102)]; long = long + df; end T = T * at; % 降温 α=0.999 if T < e, break; end end

五、Python通用实现

import numpy as np import random def simulated_annealing(f, neighbor, x0, T0=1000, alpha=0.95, L=100, T_min=1e-6): """ f: 目标函数 f(x) -> float neighbor: 邻域函数 neighbor(x) -> x_new x0: 初始解 """ x = x0 fx = f(x) best_x, best_f = x, fx T = T0 while T > T_min: for _ in range(L): x_new = neighbor(x) fx_new = f(x_new) delta = fx_new - fx if delta < 0 or np.random.random() < np.exp(-delta / T): x, fx = x_new, fx_new if fx < best_f: best_x, best_f = x, fx T *= alpha # 降温 return best_x, best_f # === TSP邻域：2-opt交换 === def tsp_neighbor(path): """随机选两点，反转中间路径""" i, j = sorted(random.sample(range(1, len(path)-1), 2)) new_path = path[:i] + path[i:j+1][::-1] + path[j+1:] return new_path

六、数学建模竞赛中的应用

适用场景：
1. TSP/VRP：旅行商、车辆路径规划
2. 调度优化：车间调度、航班调度、考试排考
3. 布局/选址：设施布局、网络节点部署
4. 参数优化：机器学习超参数调优
5. 对比基准：与遗传算法(GA)、粒子群(PSO)对比，体现算法选择的严谨性

优缺点

优点缺点
全局搜索能力强，能跳出局部最优参数敏感（T₀/α/L需精心调参）
不依赖梯度信息，适用于任意目标函数运行时间长，难以实时使用
通用性强，几乎所有组合优化问题都可用最优解不能保证，是近似算法
易于并行化（多链并行退火）冷却调度选择缺乏理论指导
