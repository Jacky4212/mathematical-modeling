8. 数据处理与数据预处理

存储论 →

一、为什么要预处理数据

1.1 现实数据的"脏"特性

当今现实世界中的数据库极易受到噪声数据、空缺数据和不一致性数据的侵扰。很多数据都是不完整的（值缺失）、含噪声的（错误的或偏离期望的孤立点）、不一致的，无法直接进行数据挖掘，或挖掘结果差强人意。

核心原则：高质量的决策来自高质量的数据。数据预处理是整个数据挖掘与知识发现过程中的一个重要步骤，在数学建模竞赛中往往是第一步也是被低估的关键一步。

1.2 变量的分类

分类维度类型说明
按取值性质连续型（定量型）在区间内可任意取值，如身高、温度
离散型（定性型）取值有限个，如性别、等级。特殊类型：周期变量（星期、月份）
按时间关系静态数据不随时间变化
动态数据（时序）随时间变化，挖掘时需要特殊预处理

二、缺失值填补

对数据挖掘的实际应用而言，即使数据量很大，具有完整数据的案例也非常少。最简单的办法是减少数据集（去掉所有有缺失值的样本），但这样会丢失大量信息。因此需要用方法填补缺失值。

2.1 均值填补法

原理

根据与含缺失值的目标属性相关性高的其他属性将样品分组，分别计算各组目标属性的均值，将各组均值作为组内所有缺失项的填补值。

优点缺点
操作简便，有效降低点估计偏差同组缺失值用同一值填补，扭曲目标属性分布；导致均值和总量估计中对方差的低估

2.2 随机填补法

原理

采用某种概率抽样的方式，从有完整信息的元组中抽取缺失数据的填补值。能够避免均值填补中填补值过于凝集的弱点，使填补值分布与真值分布更接近。

优点缺点
分布更接近真值增大了估计量的方差，稳定性不够

2.3 热卡填补法（Hot Deck Imputation）

原理

规定一个或多个排序属性，按其观察值大小对全部观察单位排序。排序属性值完全相同的观察单位称为匹配，缺失值用与之匹配的观察单位的属性值来填补。如果没有相匹配的，每次减少一个排序属性再找。

优点缺点
保持原始数据分布特征需要设计匹配逻辑，匹配不到时需要退化策略

2.4 回归填补法

原理

在现有观察值基础上，以含有缺失值的目标属性为因变量，以与目标属性相关性高的其他属性为自变量，建立最小二乘回归模型或判别模型，以估计缺失值。

注意事项：以上几种方法都存在扭曲样本分布的问题。均值填补会降低属性之间的相关关系，回归填补则会人为地加大变量之间的相关关系。竞赛中需在论文中说明选择填补方法的理由。

三、异常值（孤立点）探测

在大型数据集中，通常存在着不遵循数据模型普遍行为的样本，这些样本和其他数据有很大不同或不一致，叫做异常点。异常点可能是由测量误差造成的，也可能是数据固有的变异性结果（如信用卡欺诈检测中异常点恰是关注目标）。

3.1 一维样本异常点检测

标准差法

计算均值和标准差，设定阈值 = 均值 ± 2×标准差，区间外的数据视为潜在异常点。

示例：数据集 {3, 56, 23, 39, 156, 52, 41, 22, 9, 28, 139, 31, 55, 20, -67, 37, 11, 55, 45, 37}
均值 = 39.9，标准差 = 45.65
阈值区间 = [39.9 - 2×45.65, 39.9 + 2×45.65] = [-51.4, 131.2]
异常点：156, 139, -67（超出区间）

3.2 基于距离的异常点检测（二维以上）

原理

计算各点之间的欧氏距离 $d = \sqrt{(X_1-X_2)^2 + (Y_1-Y_2)^2}$，设定阈值距离，超出阈值的点为异常点。

3.3 基于分类预测模型的异常值探测

对总体的特征建立分类模型（正常行为特征库），针对新数据判断是否属于正常行为，发生偏离的即是异常值。常用分类器包括：决策树分类、贝叶斯分类、神经网络分类、聚类。

四、数据变换（标准化/规范化）

数据变换是将数据转换成适合于挖掘的形式。对于基于距离的方法（如神经网络、KNN、聚类），规范化可以帮助防止具有较大初始值域的属性权重过大。

4.1 最小-最大规范化

$$v' = \frac{v - \min_A}{\max_A - \min_A} \times (new\_max - new\_min) + new\_min$$

将数据映射到 [0,1] 或 [-1,1] 区间。保留了原来数据中存在的关系。但若将来遇到超过当前取值范围的数值，会引起系统出错。

4.2 Z-score规范化（零-均值规范化）

$$v' = \frac{v - \mu}{\sigma}$$

根据属性的均值和偏差来规格化。常用于属性最大值与最小值未知，或使用最大最小规范化会出现异常数据的情况。

4.3 小数定标规范化

$$v' = \frac{v}{10^j}$$

其中 j 为使 max(|v'|) < 1 的最小整数。通过移动小数点的位置来实现规范化。

五、定性（模糊）指标的量化处理

在实际问题中，很多问题涉及定性或模糊指标的定量处理，如教学质量、科研水平、工作政绩、人员素质、满意度、信誉、态度等。如何对有关问题给出定量分析？

5.1 等级量化

按国家评价标准，评价因素一般分为五个等级 A, B, C, D, E，依次对应 5, 4, 3, 2, 1。若有 A-, B+, C-, D+ 等半级，需进一步合理量化。

5.2 模糊隶属函数法

方法

构造模糊隶属函数，如偏大型柯西分布和对数函数作为隶属函数，对任何评价值给出一个合适的量化值。也可根据实际情况构造其他隶属函数，如偏大型正态分布。

核心思想：5个等级 {很满意, 满意, 较满意, 不太满意, 很不满意} → {5, 4, 3, 2, 1}。用隶属函数实现从离散评语到连续数值的平滑映射。

六、数据平滑（离散化）

一个数值型特征可能包含许多不同的值。对许多数据挖掘技术来说，这些值之间微小的区别并不重要，但可能会降低挖掘方法的性能并影响最终结果。因此，对变量的值进行平滑处理很重要。

6.1 分箱（Binning）

分箱方法通过考察"邻居"来平滑存储数据的值。存储的值被分布到一些"桶"或"箱"中，进行的是局部平滑。

三种分箱平滑方式
方法操作示例（箱：{4,8,15}）
按箱均值平滑箱内所有值替换为箱均值{9, 9, 9}
按箱中值平滑箱内所有值替换为箱中位数{8, 8, 8}
按箱边界平滑每个值替换为最近的箱边界值{4, 4, 15}

6.2 其他平滑方法

  • 回归平滑：用回归函数拟合数据来平滑
  • 聚类平滑：用聚类中心替换原始值

七、分类方法基础

7.1 什么是分类

分类的目的是学会一个分类器（分类函数或模型），该分类器能把待分类的数据映射到给定的类别中。分类可用于预测，从历史数据中自动推导出对给定数据的推广描述，从而对未来数据进行类预测。

应用场景：医疗诊断、信用卡系统的信用分级、图像模式识别等。
分类器构造方法：统计方法（贝叶斯法、非参数法）、机器学习方法（决策树法、规则归纳法）、神经网络方法、粗糙集等。

7.2 分类的两个步骤

步骤内容
步骤一：建立模型训练数据集（每个样本有类标号）→ 学习 → 分类规则/决策树/数学公式
步骤二：使用模型评估准确率 → 对未知类标号的数据进行分类

7.3 分类评价指标

指标公式含义
精确度 Accuracy$\frac{\sum TP_j}{N}$被正确分类的样本占比
查全率 Recall$\frac{TP_j}{TP_j + FN_j}$本类样本中被正确分类的比例
查准率 Precision$\frac{TP_j}{TP_j + FP_j}$分类为该类的样本中真正属于该类的比例
F-measure$\frac{(1+\beta^2) \cdot Recall \cdot Precision}{\beta^2 \cdot Recall + Precision}$查全率和查准率的加权调和平均（β通常取1）

八、决策树（ID3算法）

8.1 决策树基本概念

结构

根节点（最顶层）→ 内部节点（属性测试，每个分枝代表一个测试输出）→ 叶节点（类标号/类分布）。从根到叶的一条路径对应一条合取规则，整棵树对应一组析取表达式规则。

决策树最大的优点：学习过程中不需要使用者了解很多背景知识，只要训练例子能用属性-结论式表示出来，就能使用该算法来学习。

8.2 ID3算法 — 信息增益

ID3采用信息增益来选择能够最好地将样本分类的属性。

$$I(n_1, ..., n_m) = -\sum_{j=1}^{m} P(c_j) \log_2 P(c_j)$$

其中 $P(c_j) = n_j / total$ 为类别 $c_j$ 的先验概率。I 是分类所需的期望信息。

$$E(A) = \sum_{s=1}^{q} \frac{n_{1s} + ... + n_{ms}}{total} \cdot I(n_{1s}, ..., n_{ms})$$

E(A) 是描述属性 A 划分数据集所得的熵。熵值越小，表示属性对数据集划分的纯度越高。

$$Gain(A) = I(n_1, n_2, ..., n_m) - E(A)$$

信息增益最大的属性选为根节点，然后递归地在每个分枝上继续选择信息增益最大的属性，直到所有样本属于同一类或没有剩余属性。

8.3 ID3算法伪代码

算法 Generate_decision_tree(samples, attribute_list): // 输入：训练样本samples，候选属性集合attribute_list // 输出：一棵决策树 (1) 创建结点N (2) IF samples都在同一个类C THEN 返回N为叶结点，标记为C (3) IF attribute_list为空 THEN 返回N为叶结点，标记为samples中最普遍的类 (4) 选择attribute_list中具有最高信息增益的属性test_attribute (5) 标记结点N为test_attribute (6) FOR EACH test_attribute的已知值a_i: 由结点N长出一个条件为test_attribute=a_i的分枝 设s_i是samples中test_attribute=a_i的样本集合 IF s_i为空 THEN 加上叶结点，标记为samples中最普遍的类 ELSE 加上Generate_decision_tree(s_i, attribute_list-test_attribute)返回的结点
信息增益计算示例（买保险数据集）：
期望信息 I(9,5) = 0.940
Gain(年龄) = 0.246（最大） → 选为根节点
Gain(收入) = 0.029，Gain(信誉度) = 0.048
年龄按 ≤40 / 41~50 / >50 划分，41~50子集纯度为1（全部买保险），直接成为叶节点。

九、数学建模竞赛中的应用策略

数据处理在竞赛中的标准流程：
1. 数据审查：检查数据维度、变量类型、缺失比例、异常值情况
2. 缺失值处理：缺失<5%可删除；5%~30%用均值/回归填补；>30%考虑删除变量
3. 异常值处理：标准差法初筛 → 结合业务判断是否保留
4. 标准化：涉及距离/梯度的模型用Z-score；只需非负用Min-Max
5. 定性指标量化：等级变量直接赋值或构造隶属函数
6. 论文中说明：所有处理步骤必须在论文中说明理由
